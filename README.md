# Rocket Engine Test Bench PCB

This PCB was developed as part of my work within IPL (Innovative Propulsion Laboratory), a student association focused on the development and testing of liquid rocket engines.

## Overview
This board is part of a rocket engine test bench and is used for control and instrumentation. It interfaces with sensors (pressure, temperature, etc.) and actuators (valves, ignition systems), and is integrated into a larger embedded control architecture.

The system is built around a Teensy 4.1 microcontroller, used for real-time control, data acquisition, and communication.

## My Contribution
This work is based on a previous design that I significantly redesigned and improved.

My contributions include:
- Major redesign of the PCB architecture  
- Integration and optimization of the Teensy 4.1 within the system  
- Improvement of hardware interfaces for sensor acquisition and actuator control  
- Adaptation of the design to better fit system-level constraints and test bench integration  
- Iterative improvements based on feedback from real test campaigns  

## System Context
The PCB operates within a full test bench system including:
- embedded firmware (C/C++) for control and sequencing  
- communication interfaces (UDP-based systems)  
- supervision and data acquisition tools (Qt)  

## Engineering Context
This project was developed in a real test environment, with practical constraints related to:
- hardware reliability and safety  
- system integration  
- rapid iteration between test campaigns
